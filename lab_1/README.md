# Лабораторная работа 1
### Описание
Программная система, моделирующая работу компилятора для трансляции исходного кода на простом языке программирования в машинный код. Проект демонстрирует основные этапы компиляции: лексический анализ, синтаксичнеский анализ, оптимизация и генерация кода.

### Основные классы
Класс ```CompilerState.py ``` перечисляет все возможные состояния компилятора. Позволяет отслеживать текущий этап компиляции и управлять переходом между этапами.

Класс ```Token.py``` представляет минимальную значимую единицу исходного кода после лексического анализа. Хранит тип токена, его значение и позицию в исходном коде для точного указания места ошибки.

Класс ```ASTNode.py``` - узел абстрактного синтаксического дерева (AST). Представляет структуру программы в виде дерева, где каждый узел соответствует конструкции языка (if, while, присваивание и т.д.).

Класс ```MachineCode.py``` - контейнер для сгенерированного машинного кода. Хранит список инструкций и точку входа в программу.

Класс ```ProgrammingLanguage.py``` описывает язык программирования. Содержит ключевые слова, операторы и разделители, что позволяет анализаторам правильно интерпретировать код.

Класс ```Compiler.py``` - основной класс. Координирует работу всех компонентов и управляет состоянием.

Класс ```LexicalAnalyzer.py``` выполняет лексический анализ - разбивает исходный код на токены.

Класс ```SyntaxAnalyzer.py``` выполняет синтаксический анализ - строит AST из токенов, проверяя грамматическую правильность программы.

Класс ```Optimizer.py``` улучшает AST, делая программу эффективнее.

Класс ```MachineCodeGenerator.py``` транслирует оптимизированное AST в машинный код для целевой архитектуры.

Классы интерфейсов ```IAnalyzer```, ```IOptimizer``` и ```ICodeGenerator``` определяют контракты для основных компонентов, обеспечивая слабую связанность и возможность замены реализаций

Классы иерахии исключений.

Класс ```CLI.py``` (Интерфейс командной строки) предоставляет пользовательский интерфейс для взаимодействия с компилятором.

### Поэтапная компиляция
```
# Только лексический анализ
python -m Compiler.CLI compile test.txt --stage lexical --save-state lexical.pkl

# Синтаксический анализ (загружая предыдущее состояние)
python -m Compiler.CLI compile test.txt --stage syntax --load-state lexical.pkl --save-state syntax.pkl

# Оптимизация
python -m Compiler.CLI compile test.txt --stage optimize --load-state syntax.pkl --save-state optimized.pkl

# Генерация кода
python -m Compiler.CLI compile test.txt --stage code --load-state optimized.pkl
```

Также можно запустить меню компилятора командой:
```
python -m Compiler.CLI menu

```
### Пример входного файла
```
int x = 5;
int y = 10;
int result;

if x < y {
    result = y - x;
} else {
    result = x - y;
}

while result > 0 {
    result = result - 1;
}

return result;
```
